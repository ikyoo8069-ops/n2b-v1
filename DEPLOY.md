# 🚀 GitHub Pages 배포 가이드

## 📋 단계별 가이드

### 1단계: 새 저장소 만들기 (1분)

1. https://github.com/ikyoo8069-ops 접속
2. 오른쪽 위 **"+"** 클릭 → **"New repository"**
3. 저장소 이름 입력:
   ```
   n2b-v1
   ```
4. 설정:
   - ✅ Public (공개)
   - ✅ Add a README file (체크)
5. **"Create repository"** 클릭

---

### 2단계: 파일 업로드 (2분)

1. 방금 만든 저장소 페이지에서
2. **"Add file"** → **"Upload files"** 클릭
3. 다운로드한 n2b-v1 폴더의 **모든 파일** 드래그 앤 드롭:
   - index.html
   - css/ (폴더째)
   - js/ (폴더째)
   - data/ (폴더째)
   - README.md (선택사항)
4. 맨 아래 **"Commit changes"** 클릭

---

### 3단계: GitHub Pages 활성화 (1분)

1. 저장소 상단 **"Settings"** 클릭
2. 왼쪽 메뉴에서 **"Pages"** 클릭
3. Source 섹션:
   - Branch: **main** 선택
   - Folder: **/ (root)** 선택
4. **"Save"** 클릭
5. 5분 기다리기... ⏳

---

### 4단계: 접속! (완료)

```
https://ikyoo8069-ops.github.io/n2b-v1/
```

**→ 진짜 웹사이트 완성!** 🎉

---

## ✅ 확인사항

### 제대로 작동하는지:

1. 위 링크 접속
2. "3초 만에 시작하기" 클릭
3. "샘플로 먼저 체험하기" 클릭
4. 결과 확인:
   - 🥇 88%
   - 🥈 85%
   - 🥉 82%

**→ 50%가 아니라 80%대 나오면 성공!**

---

## 🐛 문제 해결

### Pages가 안 보이면?

1. Settings → Pages
2. "Your site is ready to be published at..." 확인
3. 5-10분 기다리기
4. 브라우저 새로고침

### 404 에러?

1. 저장소 이름 확인: n2b-v1
2. 파일 구조 확인:
   ```
   저장소 루트/
   ├── index.html
   ├── css/
   ├── js/
   └── data/
   ```
